# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability, please report it via:
- Email: security@lumadb.io
- GitHub Security Advisories (private disclosure)

**Do not** report security issues via public GitHub issues.

## Response Timeline

- Initial response: 24-48 hours
- Severity assessment: 72 hours
- Fix development: Based on severity
- Public disclosure: After fix is released

## Scope

Security issues in:
- Authentication (JWT, sessions, OAuth)
- Authorization (RLS, permissions)
- Data protection (encryption, secrets)
- Input validation (SQL injection, XSS)
- API security (rate limiting, CORS)

## Safe Harbor

We commit to not pursue legal action against good-faith security research.
